Overview

The Healthcare Diagnosis Ontology (HDO) is a formal OWL 2 DL knowledge base for clinical diagnosis support. It encodes semantic relationships between patients, symptoms, diseases, treatments, and diagnostic tests, enabling automated reasoning without explicit diagnostic assertions.

The ontology supports: (1) automated diagnosis inference via OWL 2 property chain axiom; (2) rule-based clinical decision support via SWRL; (3) semantic querying via 20 SPARQL 1.1 queries; and (4) data quality enforcement via 7 SHACL shapes (Conforms: True — zero violations across 1,925 triples).

Inference Mechanisms

OWL 2 Property Chain Axiom

reportsSymptom ∘ inverse(hasSymptom) ⊑ possibleDiagnosis

If a patient reports symptom S and a disease has symptom S, HermiT automatically infers that disease as a possible diagnosis. Example: Ali reports Cough, Fever, NightSweats → hdo:Ali hdo:possibleDiagnosis hdo:Tuberculosis (inferred, not asserted).

SWRL Clinical Rule

Patient(?p) ∧ hasSymptomDuration(?p, ?d) ∧ swrlb:greaterThan(?d, 13)
∧ hasFeverStatus(?p, "Yes")
→ requiresTest(?p, SputumTest)

Fires for: Ali (14 days, fever Yes) and Selin (45 days, fever Yes).

SPARQL Query — Q10: SWRL Rule Simulation

PREFIX hdo: <http://www.semanticweb.org/beyza/ontologies/2026/healthcare#>
SELECT ?patient ?duration ?feverStatus WHERE {
  ?p rdf:type hdo:Patient . ?p rdfs:label ?patient .
  ?p hdo:hasSymptomDuration ?duration . ?p hdo:hasFeverStatus ?feverStatus .
  FILTER (?duration > 13 && ?feverStatus = "Yes")
}
